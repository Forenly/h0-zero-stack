> Compiled from the official Devpost page. *"Front-end in minutes. Back-end designed for scale."* Build a full-stack app on **Vercel v0 + AWS Databases** that could actually go to production.

## The stack (mandatory)
Every submission, regardless of track, MUST:
1. Use **one of three designated AWS Databases:**
   - **Amazon Aurora PostgreSQL**
   - **Aurora DSQL**
   - **DynamoDB**
2. **Deploy the front end on Vercel or v0.app** (use **v0** to scaffold a production-ready Next.js front end).

Pitch: prototype over a weekend, run on the same data foundation startups/enterprises use in production.

## Tracks (pick one)
1. **Monetizable B2C app** — ecommerce, travel, retail, hospitality (side hustle → full venture).
2. **Monetizable B2B app** — finance, tech, healthcare, insurance, marketing/advertising, or any sector. ← **likely Forenly fit**
3. **Million-scale global app** — gaming, social media, entertainment; architecture designed to scale to millions of users globally.
4. **Open innovation** — anything goes; any full-stack app that creatively uses the Vercel/v0 + AWS Databases stack.

## Get started
1. Sign up for an **AWS account** (Forenly has one — root user keys in GSM; see internal AWS access note).
2. Sign up for a **v0.app** account.
3. **Fill out the request form** to get AWS & v0 credits.
4. Choose and provision your AWS Database.
5. Build. Ship. Submit.

## What to submit
1. **Text description** — state **which AWS database(s)** you used.
2. **Demo video (~3–5 min, YouTube preferred):**
   - What problem the app solves, for whom, and why this problem
   - Footage showing the **working application**
   - Explanation of the **AWS Database(s)** used
3. **Published Vercel Project Link + Vercel Team ID**
4. **Architecture diagram** — how the app connects to back-end components
5. **Screenshots of your v0/Vercel project**, including **Storage Configuration** to prove AWS Database usage

## Judging criteria
1. **Technological Implementation** — real craftsmanship; AWS DB integrated with a deliberate data model + architecture; Vercel deployment beyond basics; clean, purposeful, intentional.
2. **Design** — intuitive UX; front-end feels designed in relation to the back-end; cohesive full-stack balance.
3. **Impact & Real-world Applicability** — solves a meaningful problem for a real audience; scalable DB + frontend deployment make it shippable, not just functional.
4. **Originality** — creative, original concept; genuine insight about what's possible with this stack; if not new, how far it's pushed forward.

> Judges want **shippable software, not just demos.**
